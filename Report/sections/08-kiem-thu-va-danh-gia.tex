\section{Kiểm thử firmware}

Kiểm thử firmware tập trung vào từng driver và luồng xử lý chính. Các thư mục test hiện có gồm \texttt{testPMS7003}, \texttt{testCO2}, \texttt{testSHTC3}, \texttt{testVOC}. Mỗi test cần xác nhận cảm biến trả dữ liệu hợp lệ, timeout không làm treo hệ thống và cờ hợp lệ phản ánh đúng trạng thái kết nối.

\section{Kiểm thử tích hợp}

Source hiện tại đã có một phần kiểm thử backend, gồm unit test cho API/MQTT và integration test mô phỏng MQTT broker. Nhóm test này giúp kiểm tra contract giữa RTOS payload, MQTT service, backend API và database model ở mức phần mềm. Tuy nhiên, đây chưa thay thế cho kiểm thử end-to-end với ESP32 thật, Wi-Fi thật, Mosquitto thật và dashboard chạy cùng lúc.

Kiểm thử tích hợp cần bao phủ hai pipeline:

\begin{itemize}[leftmargin=2em]
    \item Data pipeline: \texttt{Sensor -> ESP32 -> MQTT -> Backend -> DB -> API -> Web}.
    \item Control pipeline: \texttt{Web -> API -> Backend -> MQTT -> ESP32 -> Device}.
\end{itemize}

\section{Chỉ số đánh giá}

Các chỉ số đánh giá gồm tỷ lệ mẫu đo hợp lệ, độ trễ từ lúc vượt ngưỡng đến lúc output đổi trạng thái, số lần bật/tắt trong một khoảng thời gian, tỷ lệ thời gian ở vùng an toàn, khả năng vận hành khi mất server và độ ổn định của pipeline MQTT/API/web.
