Kiến trúc tổng thể được chia thành các tầng: cảm biến, thiết bị nhúng ESP32, truyền thông MQTT, backend/API, cơ sở dữ liệu và dashboard web. Thiết bị nhúng chịu trách nhiệm đo và phản ứng cục bộ. Backend chịu trách nhiệm nhận dữ liệu từ MQTT, chuẩn hóa payload, lưu lịch sử và cung cấp API. Dashboard web là giao diện quan sát, phân tích lịch sử và gửi lệnh điều khiển.

Nguyên tắc thiết kế xuyên suốt là xử lý cục bộ trước, giám sát từ xa sau. ESP32 vẫn phải đọc cảm biến, đánh giá IAQ và điều khiển cảnh báo ngay cả khi Wi-Fi, MQTT hoặc server không khả dụng. Tầng IoT/backend/dashboard mở rộng khả năng quan sát và điều khiển từ xa, nhưng không thay thế cơ chế an toàn tại thiết bị.

\section{Luồng dữ liệu}

\begin{center}
\texttt{Sensor -> ESP32/Firmware -> MQTT -> Backend -> Database -> API -> Web Dashboard}
\end{center}

Luồng điều khiển đi ngược lại:

\begin{center}
\texttt{Web Dashboard -> API -> Backend -> MQTT -> ESP32 -> Device/LED}
\end{center}

Trong source hiện tại, đoạn \texttt{Sensor -> ESP32/Firmware -> log/LED} đã được hiện thực ở nhánh firmware tuần tự. Phiên bản RTOS mở rộng luồng này bằng MQTT để kết nối với backend, database và dashboard.

Với phiên bản RTOS hiện tại, luồng dữ liệu đã được hiện thực ở mức prototype qua MQTT:

\begin{center}
\texttt{SensorHub -> RTOS tasks -> MQTT -> Backend -> MySQL/API -> Dashboard}
\end{center}

Luồng điều khiển từ xa được thiết kế theo hướng:

\begin{center}
\texttt{Dashboard -> API -> MQTT control topic -> RTOS control task -> IaqController}
\end{center}

\section{Nguyên tắc thiết kế}

Các tầng được tách theo trách nhiệm. Driver chỉ đọc cảm biến và kiểm tra giao thức. \texttt{SensorHub} gom dữ liệu thành \texttt{SensorSample}. \texttt{IaqEvaluator} chỉ xử lý trạng thái IAQ, không phụ thuộc chân GPIO. \texttt{IaqController} chỉ áp dụng output. Trong phiên bản RTOS, dữ liệu được truyền giữa task qua queue, mutex và semaphore. Backend và dashboard không đọc trực tiếp phần cứng, mà làm việc qua MQTT/API theo contract chung.
